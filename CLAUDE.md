# ARID

## Changelog discipline

Whenever you make a change to this repo — code, docker/config, pipeline
behavior, performance work, etc. — append an entry to `CHANGELOG.md`
summarizing what changed and why. Add it under today's date (create a new
dated section if one doesn't exist yet). Do this regardless of which Claude
interface or session you're running in; the changelog is the durable record,
not any one conversation.
